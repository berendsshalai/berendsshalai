# Emerald observatory glass system

The GitHub profile SVGs use one deterministic, self-contained optical material generated by `scripts/generate_optical_glass_assets.py`.

The SVG canvas remains transparent. Each pane is formed from restrained native SVG layers: two quiet shadows, a green-biased transmission field, a feathered side-depth layer, broad internal illumination, paired refractive edges and isolated asymmetric specular highlights. A focused emerald light completes one subtle circuit around the outer border every nine seconds and disappears when reduced motion is requested. Foreground text and icons are emitted after the material and are never blurred.

The identity system uses GitHub green (`#3FB950`) and a three-level type hierarchy: Space Grotesk for architectural display text, Inter for readable profile information, and IBM Plex Mono for telemetry. Theme-aware SVG rules preserve contrast in GitHub light and dark modes.

No pane uses raster backgrounds, embedded images, JavaScript, external CSS, `foreignObject` or backdrop filtering. Compact information cards reuse the same `chipGlass` and `chipEdge` definitions.

Regenerate and validate with:

```bash
python scripts/generate_optical_glass_assets.py
python scripts/validate_optical_glass.py
python scripts/generate_repository_globe.py
python scripts/validate_repository_globe.py
```

For rendered alpha validation, render each target SVG to a same-named PNG and pass its directory:

```bash
python scripts/validate_optical_glass.py --rendered-dir path/to/rendered
```
